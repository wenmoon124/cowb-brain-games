# 国际化指南

本文档详细说明 BrainVerse 项目（cowb.cc）的国际化（i18n）架构，涵盖多语言架构概述、支持语言、路由规则、翻译文件组织、翻译 key 规范、添加新语言流程、内容多语言以及本地化原则。

> **核心原则**：所有面向用户的文本必须通过 i18n key 获取，严禁硬编码文本。代码中不得出现任何用户可见的字符串字面量（除占位符与变量外）。

---

## 1. 多语言架构概述

### 1.1 技术方案

BrainVerse 基于 Next.js 14 App Router 实现国际化，采用动态路由段 `[locale]` 携带语言信息：

| 方案 | 说明 |
|---|---|
| **路由方案** | App Router 动态段 `[locale]`，URL 携带语言前缀（`/en/...`、`/zh/...`、`/ja/...`） |
| **翻译加载** | 使用 `next-intl` 或 `next-i18n-router` 加载 JSON 翻译文件 |
| **翻译文件** | JSON 格式，按命名空间组织（`apps/web/i18n/en.json`、`zh.json`、`ja.json`） |
| **语言切换** | 客户端组件 `LanguageSwitcher`，切换时保持当前路径 |
| **SEO 配合** | 通过 hreflang alternate links 声明多语言版本（详见 [seo.md](./seo.md)） |

### 1.2 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    Next.js App Router                        │
│                                                              │
│  app/                                                       │
│  ├── [locale]/                ← 动态语言段                   │
│  │   ├── layout.tsx           ← 验证 locale、加载翻译        │
│  │   ├── page.tsx             ← 首页（按 locale 渲染）        │
│  │   ├── games/                                              │
│  │   ├── articles/                                           │
│  │   └── dashboard/                                          │
│  ├── layout.tsx               ← 根布局                       │
│  ├── sitemap.ts               ← 多语言 sitemap               │
│  └── robots.ts                                              │
│                                                              │
│  i18n/                                                     │
│  ├── en.json                  ← 英语翻译                    │
│  ├── zh.json                  ← 简体中文翻译                │
│  ├── ja.json                  ← 日语翻译                    │
│  └── config.ts                ← i18n 配置                   │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 为什么选择路由段方案

| 方案 | 优点 | 缺点 | BrainVerse 选择 |
|---|---|---|---|
| **路由段 `/[locale]/...`** | SEO 友好、URL 明确、可缓存 | URL 较长 | ✅ 采用 |
| Cookie/Session 检测 | URL 简洁 | SEO 不友好、不可缓存 | ❌ 不采用 |
| 子域名 `en.cowb.cc` | SEO 友好 | DNS 配置复杂、Cloudflare 免费计划限制 | ❌ 不采用 |

## 2. 支持语言

### 2.1 语言清单

BrainVerse 支持三种语言：

| 语言代码 | 语言名称 | Locale 标识 | 字符集 | 默认 |
|---|---|---|---|---|
| `en` | English | `en-US` | Latin | ✅ 默认语言 |
| `zh` | 简体中文 | `zh-CN` | 中文 | ❌ |
| `ja` | 日本語 | `ja-JP` | 仮名 + 漢字 | ❌ |

### 2.2 语言选择理由

- **English (`en`)**：全球通用语言，覆盖最广用户群体，作为默认与 x-default
- **简体中文 (`zh`)**：覆盖中国大陆及华语圈，潜在用户规模最大
- **Japanese (`ja`)**：日本是全球脑训练市场成熟地区（任天堂脑トレ文化），有明确需求

### 2.3 语言配置

```typescript
// apps/web/i18n/config.ts
export const LOCALES = ['en', 'zh', 'ja'] as const
export type Locale = (typeof LOCALES)[number]

export const DEFAULT_LOCALE: Locale = 'en'

export const LOCALE_NAMES: Record<Locale, string> = {
  en: 'English',
  zh: '简体中文',
  ja: '日本語',
}

export const LOCALE_FLAGS: Record<Locale, string> = {
  en: '🇺🇸',
  zh: '🇨🇳',
  ja: '🇯🇵',
}

// HTML lang 属性值
export const LOCALE_HTML_LANG: Record<Locale, string> = {
  en: 'en',
  zh: 'zh-CN',
  ja: 'ja',
}

// OpenGraph locale 值
export const LOCALE_OG: Record<Locale, string> = {
  en: 'en_US',
  zh: 'zh_CN',
  ja: 'ja_JP',
}

// 验证 locale 是否有效
export function isValidLocale(locale: string): locale is Locale {
  return LOCALES.includes(locale as Locale)
}
```

## 3. 路由规则

### 3.1 URL 结构

所有页面 URL 必须以 locale 前缀开头：

```
/en                    ← 英文首页
/en/games              ← 英文游戏中心
/en/articles/brain-age/what-is-brain-age
/zh                    ← 中文首页
/zh/games
/ja                    ← 日文首页
/ja/games
```

### 3.2 根路径重定向

访问根路径 `/` 时重定向到默认语言 `/en`：

```javascript
// apps/web/next.config.js
module.exports = {
  async redirects() {
    return [
      {
        source: '/',
        destination: '/en',
        permanent: false,  // 临时重定向，便于未来语言检测
      },
      {
        source: '/games',
        destination: '/en/games',
        permanent: false,
      },
      {
        source: '/dashboard',
        destination: '/en/dashboard',
        permanent: false,
      },
      // 其他无 locale 前缀的路径重定向
    ]
  },
}
```

### 3.3 locale 验证

`[locale]/layout.tsx` 中验证 locale 参数，无效 locale 返回 404：

```typescript
// apps/web/app/[locale]/layout.tsx
import { notFound } from 'next/navigation'
import { LOCALES, isValidLocale, DEFAULT_LOCALE } from '@/i18n/config'
import type { Locale } from '@brainverse/shared'

// 预生成所有 locale 路径
export function generateStaticParams() {
  return LOCALES.map(locale => ({ locale }))
}

export default async function LocaleLayout({
  children,
  params: { locale },
}: {
  children: React.ReactNode
  params: { locale: string }
}) {
  // 验证 locale
  if (!isValidLocale(locale)) {
    notFound()  // 无效 locale 返回 404
  }

  // 加载翻译
  const messages = (await import(`@/i18n/${locale}.json`)).default

  return (
    <html lang={LOCALE_HTML_LANG[locale]}>
      <body>
        <NextIntlClientProvider locale={locale} messages={messages}>
          <Navbar />
          {children}
          <Footer />
          <CookieConsent />
        </NextIntlClientProvider>
      </body>
    </html>
  )
}
```

### 3.4 语言切换保持路径

语言切换时保持当前路径，仅替换 locale 前缀：

```typescript
// apps/web/components/layout/language-switcher.tsx
'use client'

import { usePathname, useRouter } from 'next/navigation'
import { LOCALES, LOCALE_NAMES, type Locale } from '@/i18n/config'
import { useState } from 'react'

export function LanguageSwitcher({ currentLocale }: { currentLocale: Locale }) {
  const router = useRouter()
  const pathname = usePathname()
  const [open, setOpen] = useState(false)

  function switchLocale(newLocale: Locale) {
    // 替换路径中的 locale 前缀
    const segments = pathname.split('/')
    segments[1] = newLocale  // segments[0] 为空字符串
    const newPath = segments.join('/') || `/${newLocale}`
    router.push(newPath)
    setOpen(false)
  }

  return (
    <div className="relative">
      <button onClick={() => setOpen(!open)} aria-label="Switch language">
        {LOCALE_NAMES[currentLocale]}
      </button>
      {open && (
        <ul className="absolute right-0 mt-2 bg-white shadow-lg">
          {LOCALES.map(locale => (
            <li key={locale}>
              <button onClick={() => switchLocale(locale)}>
                {LOCALE_NAMES[locale]}
              </button>
            </li>
          ))}
        </ul>
      )}
    </div>
  )
}
```

### 3.5 路由规则总结

| 访问 URL | 行为 |
|---|---|
| `/` | 重定向到 `/en` |
| `/en` | 显示英文首页 |
| `/zh` | 显示中文首页 |
| `/ja` | 显示日文首页 |
| `/fr` | 返回 404（不支持的 locale） |
| `/games` | 重定向到 `/en/games` |
| `/en/articles/brain-age/what-is-brain-age` | 显示英文文章 |
| `/zh/articles/brain-age/what-is-brain-age` | 显示中文文章（同一文章的中文翻译） |

### 4.4 翻译文件同步

**核心规则：三个翻译文件（en/zh/ja）的 key 结构必须完全一致**。

- 新增 key → 三个文件同时新增
- 删除 key → 三个文件同时删除
- 修改 key → 三个文件同时修改
- 检查工具：CI 中运行翻译完整性检查脚本，对比 key 结构

```typescript
// scripts/check-i18n.ts（翻译完整性检查示例）
import en from '../apps/web/i18n/en.json'
import zh from '../apps/web/i18n/zh.json'
import ja from '../apps/web/i18n/ja.json'

function getKeys(obj: Record<string, unknown>, prefix = ''): string[] {
  return Object.entries(obj).flatMap(([key, value]) => {
    const fullKey = prefix ? `${prefix}.${key}` : key
    if (value && typeof value === 'object') {
      return getKeys(value as Record<string, unknown>, fullKey)
    }
    return [fullKey]
  })
}

const enKeys = getKeys(en).sort()
const zhKeys = getKeys(zh).sort()
const jaKeys = getKeys(ja).sort()

const missingInZh = enKeys.filter(k => !zhKeys.includes(k))
const missingInJa = enKeys.filter(k => !jaKeys.includes(k))
const extraInZh = zhKeys.filter(k => !enKeys.includes(k))
const extraInJa = jaKeys.filter(k => !enKeys.includes(k))

if (missingInZh.length || missingInJa.length || extraInZh.length || extraInJa.length) {
  console.error('i18n key mismatch:')
  if (missingInZh.length) console.error('Missing in zh.json:', missingInZh)
  if (missingInJa.length) console.error('Missing in ja.json:', missingInJa)
  if (extraInZh.length) console.error('Extra in zh.json:', extraInZh)
  if (extraInJa.length) console.error('Extra in ja.json:', extraInJa)
  process.exit(1)
}

console.log('✓ All i18n files have matching keys')
```

## 5. 翻译 key 规范

### 5.1 key 命名格式

翻译 key 使用 **dot notation**（点号分隔），层级反映命名空间结构：

```
namespace.category.key
```

### 5.2 key 命名规范

| 规范 | 说明 | 示例 |
|---|---|---|
| **全小写** | key 全部小写 | `common.appName` ✅ / `common.AppName` ❌ |
| **camelCase** | 多单词使用 camelCase | `common.buttons.learnMore` ✅ / `common.buttons.learn_more` ❌ |
| **语义化** | key 名描述用途，非内容 | `nav.home` ✅ / `nav.link1` ❌ |
| **不重复** | 同一命名空间内 key 不重复 | - |

### 5.4 在代码中使用翻译 key

#### 5.4.1 Server Component（服务端组件）

```tsx
// apps/web/app/[locale]/page.tsx
import { useTranslations } from 'next-intl'  // 或对应库的 server 端 API

export default async function HomePage({ params: { locale } }) {
  const t = await getTranslations(locale, 'landing')

  return (
    <main>
      <h1>{t('hero.title')}</h1>
      <p>{t('hero.subtitle')}</p>
      <button>{t('hero.cta')}</button>
    </main>
  )
}
```

#### 5.4.2 Client Component（客户端组件）

```tsx
// apps/web/components/layout/navbar.tsx
'use client'

import { useTranslations } from 'next-intl'

export function Navbar() {
  const t = useTranslations('nav')

      <Link href="/en/games">{t('games')}</Link>
      <Link href="/en/dashboard">{t('dashboard')}</Link>

```

### 5.5 带变量的翻译

翻译值支持变量插值，使用 `{variableName}` 语法：

```json
// en.json
{
  "dashboard": {
    "welcome": "Welcome back, {name}",
    "totalScore": "Total Score: {score}"
  },
  "games_runtime": {
    "level": "Level {n}",
    "accuracy": "Accuracy: {percent}%"
  }
}
```

```tsx
const t = useTranslations('dashboard')
const welcomeText = t('welcome', { name: 'Alice' })
// 输出："Welcome back, Alice"

const t2 = useTranslations('games_runtime')
const levelText = t2('level', { n: 5 })
// 输出："Level 5"
```

### 5.6 严禁硬编码文本

**核心规则：代码中不得出现任何用户可见的字符串字面量**。

```tsx
// ❌ 禁止：硬编码文本
<h1>Welcome to BrainVerse</h1>
<button onClick={handleClick}>Start Game</button>
<p>Please enter your email</p>

// ✅ 正确：使用翻译 key
<h1>{t('landing.hero.title')}</h1>
<button onClick={handleClick}>{t('common.buttons.start')}</button>
<p>{t('auth.signIn.email')}</p>
```

#### 例外情况

以下情况允许硬编码字符串：

1. **占位符属性**：`<input placeholder="example@cowb.cc" />`（如非用户可见 UI 文案）
2. **代码注释**：`// This function calculates brain age`
3. **控制台日志**：`console.error('Database connection failed')`
4. **API 响应**：`return { error: 'INVALID_TOKEN' }`（错误代码，非用户可见）
5. **测试断言**：`expect(result).toBe('expected value')`

### 5.7 HTML lang 属性

`<html lang>` 属性必须随 locale 正确变化：

```tsx
// apps/web/app/[locale]/layout.tsx
import { LOCALE_HTML_LANG } from '@/i18n/config'

export default function LocaleLayout({ params: { locale } }) {
  return (
    <html lang={LOCALE_HTML_LANG[locale]}>
      <body>{children}</body>
    </html>
  )
}
```

| Locale | lang 属性 |
|---|---|
| `en` | `lang="en"` |
| `zh` | `lang="zh-CN"` |
| `ja` | `lang="ja"` |

## 6. 添加新语言流程

当需要添加新语言（如 Korean `ko`）时，按以下流程操作：

### 6.1 Step 1：更新 i18n 配置

```typescript
// apps/web/i18n/config.ts
export const LOCALES = ['en', 'zh', 'ja', 'ko'] as const  // 添加 'ko'
export type Locale = (typeof LOCALES)[number]

export const LOCALE_NAMES: Record<Locale, string> = {
  en: 'English',
```

## 7. 内容多语言

### 7.1 文章多语言架构

BrainVerse 的文章通过 `article_translations` 表支持多语言：

```sql
-- articles 表（文章主表，与语言无关）
CREATE TABLE articles (
  id TEXT PRIMARY KEY,
  slug TEXT NOT NULL UNIQUE,
  category TEXT NOT NULL,
  published INTEGER NOT NULL DEFAULT 0,
  featured INTEGER NOT NULL DEFAULT 0,
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

-- article_translations 表（按语言存储文章内容）
CREATE TABLE article_translations (
  id TEXT PRIMARY KEY,
  article_id TEXT NOT NULL,
  locale TEXT NOT NULL,
  title TEXT NOT NULL,
  meta_description TEXT,
  content TEXT NOT NULL,
  faq TEXT,  -- JSON 格式存储 FAQ
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (article_id) REFERENCES articles(id),
  UNIQUE(article_id, locale)  -- 一篇文章一个语言一条记录
);
```

### 7.2 三语版本要求

**核心规则：文章必须支持 en/zh/ja 三语版本，三语版本齐全方可发布**。

- `articles.published = 1`（已发布）的前提是该文章的 `article_translations` 中存在 en、zh、ja 三条记录
- 文章发布前由系统或 Admin 检查三语版本完整性
- 如某语言版本缺失，文章保持 `published = 0`（草稿状态）

### 7.3 三语版本一致性

虽然翻译应本地化（非直译），但三语版本必须保持：

| 维度 | 一致性要求 |
|---|---|
| **slug** | 三语版本使用相同 slug（URL 不区分语言内容） |
| **category** | 三语版本属于相同类别 |
| **科学引用** | 三语版本引用相同的文献（文献作者、年份、期刊一致） |
| **关联游戏** | 三语版本关联相同的平台游戏 |
| **结构** | 三语版本遵循统一 8 段结构 |
| **FAQ 数量** | 三语版本 FAQ 条数一致 |
| **内链目标** | 三语版本内链指向相同文章（不同语言路径） |

### 7.4 内容查询

API 层根据请求的 locale 返回对应语言的内容：

```typescript
// workers/api/src/db/repositories/article-repo.ts
export class ArticleRepository {
  /**
   * 根据 slug 和 locale 获取已发布文章（含翻译）
   */
  async findBySlug(slug: string, locale: Locale): Promise<ArticleWithTranslation | null> {
    const result = await this.db
      .prepare(
        `SELECT
           a.id, a.slug, a.category, a.published, a.featured,
           t.title, t.meta_description, t.content, t.faq,
           t.created_at, t.updated_at
         FROM articles a
         JOIN article_translations t ON a.id = t.article_id
         WHERE a.slug = ? AND t.locale = ? AND a.published = 1`
      )
      .bind(slug, locale)
      .first<ArticleRow>()

    if (!result) return null

    return {
      id: result.id,
      slug: result.slug,
      category: result.category,
      title: result.title,
      metaDescription: result.meta_description,
      content: result.content,
      faq: result.faq ? JSON.parse(result.faq) : null,
      published: result.published === 1,
      featured: result.featured === 1,
    }
  }
}
```

## 8. 本地化原则

### 8.1 本地化 vs 直译

**核心原则：翻译必须本地化（localization），而非直译（literal translation）**。

直译只是字面转换，本地化则考虑目标语言的文化背景、表达习惯、示例适配，使内容对目标语言用户自然、亲切。

### 8.2 本地化要求

#### 8.2.1 语言表达本地化

- **避免翻译腔**：不生搬硬套英语句式，使用目标语言的自然表达
- **符合语言习惯**：日语使用适当的敬语、中文使用符合现代汉语习惯的表述
- **术语统一**：专业术语在三语中保持一致的翻译（建立术语表）

#### 8.2.2 文化适配

- **示例本地化**：将文化相关示例替换为目标语言文化中的对应物
  - 英语："like a baseball player tracking a fastball" → 日语：「野球选手追踪直球」
  - 英语文化典故 → 中文/日文文化对应
- **计量单位**：使用目标地区常用单位（如公制/英制）
- **日期格式**：使用目标语言习惯的日期格式

#### 8.2.3 内容适配

- **研究引用**：科学文献引用保留原文（作者、年份、期刊名），但说明文字本地化
- **游戏关联**：关联游戏名称使用本地化翻译
- **图片**：如含文字的图片，需提供对应语言版本

### 8.3 本地化示例

#### 示例 1：UI 文案

```
// en.json
"dashboard": {
  "welcome": "Welcome back, {name}"
}

// zh.json（本地化）
"dashboard": {
  "welcome": "欢迎回来，{name}"
}

// ja.json（本地化，注意敬语）
"dashboard": {
  "welcome": "おかえりなさい、{name}さん"
}
```

#### 示例 2：文章内容

```
// English (en)
"As Dr. Walker notes in his 2017 book 'Why We Sleep', 
slow-wave sleep is critical for memory consolidation. 
Think of it like saving a document on your computer—
without hitting 'Save,' your work might be lost."

// Chinese (zh) — 本地化翻译
"正如 Walker 博士在 2017 年著作《我们为什么睡觉》中所述，
慢波睡眠对记忆巩固至关重要。这就像在电脑上保存文档——
如果不点击'保存'，你的工作可能会丢失。"

// Japanese (ja) — 本地化翻译，注意敬体
"Walker 博士は 2017 年の著書『Why We Sleep』の中で、
徐波睡眠が記憶の定着に不可欠であると述べています。
パソコンで文書を保存するのと同じで、
「保存」を押さないと作業内容が失われてしまう可能性があります。"
```

#### 示例 3：文化示例适配

```
// English (en)
"Like a quarterback scanning the field for open receivers, 
your brain's selective attention filters out distractions."

// Chinese (zh) — 替换为更熟悉的运动比喻
"就像乒乓球选手在快速对拉中专注判断球路一样，
你的大脑选择性注意力会过滤掉周围的干扰。"

// Japanese (ja) — 替换为日本熟悉的运动比喻
"野球のバッターが投球を見極めるように、
脳の選択的注意は周囲の雑音をフィルタリングします。"
```

### 8.4 本地化质量保证

- **母语审校**：中文翻译由中文母语者审校，日语翻译由日语母语者审校
- **术语表**：建立三语术语对照表，确保专业术语一致
- **风格指南**：为每种语言制定风格指南（语气、用词偏好、格式规范）
- **用户反馈**：收集目标语言用户反馈，持续优化翻译质量

### 8.5 日期与数字本地化

使用 `Intl` API 根据 locale 格式化日期和数字：

```typescript
// apps/web/lib/utils.ts
export function formatDate(date: Date | string, locale: Locale): string {
  const d = typeof date === 'string' ? new Date(date) : date
  return new Intl.DateTimeFormat(locale, {
    year: 'numeric',
    month: 'long',
    day: 'numeric',
  }).format(d)
}

export function formatNumber(n: number, locale: Locale): string {
  return new Intl.NumberFormat(locale).format(n)
}

export function formatPercent(n: number, locale: Locale): string {
  return new Intl.NumberFormat(locale, {
    style: 'percent',
    maximumFractionDigits: 1,
  }).format(n)
}
```

#### 格式化示例

| Locale | 日期 | 数字 | 百分比 |
|---|---|---|---|
| `en` | June 25, 2026 | 12,500 | 85.5% |
| `zh` | 2026年6月25日 | 12,500 | 85.5% |
| `ja` | 2026年6月25日 | 12,500 | 85.5% |

### 8.6 复数处理

不同语言的复数规则不同，使用 i18n 库的复数支持：

```json
// en.json
{
  "dashboard": {
    "streakDays": "{count, plural, one {# day} other {# days}}"
  }
}

// zh.json（中文无复数区分）
{
  "dashboard": {
    "streakDays": "{count} 天"
  }
}

// ja.json（日语无复数区分）
{
  "dashboard": {
    "streakDays": "{count} 日"
  }
}
```
